Methodology

Pipeline Overview

End-to-end ML pipeline from raw network traffic to explainable predictions.

1

Data Preprocessing

Load CIC-IDS-2017 CSV files, clean infinite/NaN/duplicate values, encode 15 traffic classes, standardize 78 features.

2

Model Training

Train Logistic Regression, Random Forest, and XGBoost with stratified 70/10/20 train/val/test splits.

3

Evaluation

Compute accuracy, precision, recall, F1-score. Generate confusion matrices and model comparison charts.

4

Explainability

SHAP for global feature importance. LIME for local instance-level explanations of correct and misclassified samples.

Limitations & Known Issues

We believe in transparency. Here's what the project can and cannot do.

⚠️ Synthetic Data Default

All published results are from synthetic data (~50K samples). The real CIC-IDS-2017 dataset (2.8M+ samples) requires downloading via --download flag. Real-data benchmark results are pending.

❌ Class Imbalance

5 attack types have very low recall: SQL Injection (0%), DoS Slowhttptest (3%), DoS slowloris (4%), Heartbleed (20%), DoS GoldenEye (12%). This is due to severe class imbalance (600:1 ratio). Class weighting improved macro F1 from 0.44 to 0.51.

📊 Metrics Transparency

Weighted F1 (0.82) appears strong but hides minority class failures. Macro F1 (0.51) provides a fairer evaluation. Always check per-class metrics, not just aggregate scores.

✅ What's Working

Cross-validation, PR curves, calibration curves, failure analysis, Docker, FastAPI, and comprehensive tests are all implemented. See README for CLI usage.
