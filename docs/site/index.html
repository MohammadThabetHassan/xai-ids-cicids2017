Benchmark Datasets

Three intrusion detection datasets with varying complexity.

01

CIC-IDS-2017

14 classes (1 benign + 13 attacks). 78 features, 2.8M+ records. DoS, DDoS, Bot, Web attacks.

02

UNSW-NB15

10 classes. 45 features, 2.5M+ records. Analysis, Backdoor, Exploits, Fuzzers, Worms.

03

CSE-CIC-IDS-2018

Binary classification. 80 features, 10M+ records. Benign vs DDoS attacks-LOIC-HTTP.

04

Evaluation

20 features selected per dataset by importance. SMOTE for imbalance. XGBoost primary model.

Novel Metric

XAI Confidence Score (XCS)

Measuring the reliability of XAI explanations.

XCS Formula

XCS = 0.4 × Confidence + 0.3 × (1 - SHAP_Instability) + 0.3 × Jaccard(SHAP, LIME)

Confidence (40%): Model's prediction probability
SHAP Instability (30%): Feature ranking consistency under perturbation
Jaccard (30%): Agreement between SHAP and LIME top-k features

Cross-dataset Jaccard similarity: 0.216 — indicating dataset-specific feature patterns.

Limitations

What this project can and cannot do.

✓ Multi-Dataset Evaluation

Evaluated on 3 real IDS datasets. XGBoost achieves 99.66% (CICIDS2017), 80% (UNSW-NB15), 99.9% (CICIDS2018).

⚠ Class Imbalance

UNSW-NB15 minority classes (Analysis, Backdoor, Shellcode, Worms) have lower detection rates (~40-50% recall).

◈ Cross-Dataset Generalization

Cross-dataset Jaccard = 0.216. Models trained on one dataset may not generalize well to others.

✓ XCS Implementation

Novel XCS metric implemented to measure explanation reliability. XCS > 0.3 indicates acceptable confidence.
